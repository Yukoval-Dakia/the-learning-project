# 数学计算题出题规范

## 结构描述符（这类题落在哪三维坐标上）

> 先按 `_shared/quiz-gen` 的「题型 = 结构描述符」框架定位本规范包覆盖的题形，再套用下面的样板。

- **嵌套**：**单题（flat）**——一份条件完备的题面 + 一份可核查解答。
- **排版**：「条件 → 所求」，公式用 LaTeX（`renderConfig.notation='katex'`）；`reference_md` 给完整可核查的推导链（列条件 → 写依据 → 留关键中间式 → 得最终答案 + 检验）。
- **答案语义**：默认**受限（exact）**——只验最终答案，且因常有多种等价书写需给 `final_answer` + `answer_equivalents`；若要验方法/步骤要点则该小项走 semantic + `required_points`。

本规范包就是为这组坐标（flat / 条件→所求 + 可核查推导 / exact 最终答案 + 等价形式）写的样板。它在数学（math）学科把上述结构具体化为「在给定条件下，按正确方法推导出确定数值/表达式结果」的练习题：一道合格的题必须**条件完备、答案唯一确定、解题步骤可核查**。本规范定义题面、解答、答案格式标准，供出题（QuizGenTask）和质检（QuizVerifyTask 的 kind_conformance + solve-check）共用——solve-check 会用独立 solver 真解一遍，所以题必须真的可解、答案必须真的对。

## 题面条件完备性（核心）

计算题最常见的坏题成因是**条件不全或自相矛盾**。出题时必须保证：

1. **条件充分**：题面给出的已知量足以唯一确定所求。不能缺关键条件（如求三角形面积却没给够边/角）。
2. **条件相容**：各条件之间不矛盾（如同时给出无法共存的取值）。
3. **所求明确**：清楚说明「求什么」（具体的值 / 取值范围 / 表达式），单位 / 定义域 / 精度要求若有则写明。
4. **无歧义**：变量、符号、记号定义清楚；用 LaTeX 写公式（`renderConfig.notation='katex'`）。

## 解题步骤规范

`reference_md` 必须给出**完整可核查的解题过程**，不只给答案：

1. **列条件 / 设未知**：把题面条件形式化，必要时设变量。
2. **写依据**：每一步变形标明用了哪个定义 / 公式 / 定理（如「由韦达定理」「由正弦定理」）。
3. **保留关键中间式**：不跳步到答案；中间结果要能让人逐步验证。
4. **得出最终答案**：明确标出最终结果，并做必要的检验（如验根、检查定义域、量纲/单位一致）。

## 答案格式与判分

- `judge_kind_override`：只验**最终答案**用 `exact`；若要验方法/步骤要点用 `semantic` + `required_points`。
- 计算题的最终答案常有**多种等价书写**（如 `1/2` 与 `0.5`、`x=2 或 x=-1` 与 `{-1, 2}`）。出题时应在结构化答案里给出 `final_answer` + `answer_equivalents`（等价形式列表），让 solve-check / judge 的 normalize 比对能命中。
- 选择题形式的计算题：给 4 个选项，`reference_md` 第一行是正确选项原文，干扰项设在常见计算/方法错误上（符号错、漏解、定义域忽略）。

## 可解性与 solve-check 契合

- 题必须**真的可解且答案真的对**：质检会用独立 solver 重解，solver 答案与题面 `reference_md` / `final_answer` 不一致即 solve-check fail。
- 因此不要出「参考答案算错」的题，也不要出依赖题面外隐含约定才有唯一解的题。

## 难度分级

- 难度 1–2：单步 / 双步直接套公式。
- 难度 3–4：需选择方法 + 多步推导，含一个易错点（定义域 / 分类讨论 / 符号）。
- 难度 5：综合多个知识点，需较长推导链或巧妙变形。
