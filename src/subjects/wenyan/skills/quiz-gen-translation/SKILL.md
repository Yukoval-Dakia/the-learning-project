# 文言文翻译题出题规范

## 结构描述符（这类题落在哪三维坐标上）

> 先按 `_shared/quiz-gen` 的「题型 = 结构描述符」框架定位本规范包覆盖的题形，再套用下面的样板。

- **嵌套**：**单题（flat）**——一段需要转换的原文 + 一份参考转换结果。
- **排版**：「原文句 + 转换指令（标出重点译出的字词）+ 采分点说明」三件。
- **答案语义**：**开放（semantic）**——译文是开放表述，不能逐字 exact 比对，靠采分点（`required_points`）核查是否踩中得分要点。

本规范包就是为这组坐标（flat / 原文+指令+采分点 / semantic）写的样板。它在文言文（wenyan）学科把上述结构具体化为「把一段文言原文准确、通顺地译成现代汉语」的练习题：一道合格的题不是「给一句古文让学生随便译」，而是一道**采分点明确、参考译文可核查**的结构化练习题。本规范定义题面、参考答案、采分点的标准形状，供出题（QuizGenTask / SourcingTask）和质检（QuizVerifyTask 的 kind_conformance 检查）共用同一份标准。

## 题面结构要求

一道翻译题必须包含三个部分：

1. **原文句**：一段或一句需要翻译的文言原文（通常 1–3 句，20–60 字）。原文必须是**真实存在的古文**（出自典籍 / 课内篇目 / 真题），不得自造伪文言。若题面给了上下文，原文句要在上下文中可定位。
2. **翻译指令**：明确「把下面的句子译成现代汉语」，并标出**需要重点译出的字词**（即采分点）。
3. **参考译文 + 采分点说明**（落在 `reference_md`）：给出一个准确通顺的现代汉语译文，并逐条列出本题的采分点（每个采分点对应一个关键实词 / 虚词 / 句式 / 活用，说明它为什么是得分点、正确译法是什么）。

## 采分点设计（核心）

翻译题的质量取决于采分点是否抓住了**该句真正的文言难点**。每道题应设 2–4 个采分点，覆盖以下至少两类：

- **关键实词**：古今异义（如「妻子」=妻子儿女）、一词多义（如「兵」=兵器/士兵/战争）、通假字。
- **虚词功能**：「之/其/以/而/乃/为」等在该句的具体语法功能（取消句子独立性、宾语前置标志、介词等）。
- **特殊句式**：判断句、被动句、宾语前置、状语后置、省略句 —— 译文必须把句式还原成现代汉语语序。
- **词类活用**：名词作动词、使动、意动、形容词作名词等。

采分点必须**可核查**：参考译文里要能逐一对应到每个采分点的正确译法，质检据此判断学生译文是否踩中得分点。

## 答案与判分格式

- `judge_kind_override` 应为 `semantic`（翻译是开放表述，不能 exact 比对）。
- `rubric_json.required_points` 必填：每个采分点写成一条可核查要点（如「『卒』译为『最终/终于』，不可译为『士兵』」）。required_points 的条数应与采分点数一致（2–4 条）。
- `reference_md` 第一段给完整参考译文，其后逐条列采分点。

## 难度分级

- 难度 1–2：课内常见句，采分点多为高频实词 / 基础句式。
- 难度 3–4：含 1–2 个易混活用或特殊句式，或古今异义陷阱。
- 难度 5：长句 + 多重句式嵌套，或生僻实词需结合语境推断。
