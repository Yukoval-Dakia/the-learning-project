# Math MVP — M-1 + M0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `question` 表升级为多模态 first-class 承载，端到端跑通 math (choice + fill_blank) 10 道 fixture 一轮答题；同时验证 xiaomi/mimo endpoint vision 能力，输出 drift 探针清单为 M1 准备。

**Architecture:**
- M-1: Migration 0010 给 `question` 表加 `figures` / `image_refs` / `structured` 三个 jsonb 字段；改造 `app/api/ingestion/[id]/import/route.ts` 把 question_block 的多模态结构传递到 question 表（不再仅靠 metadata.prompt_image_refs）；`JudgeQuestionRow` 扩展含新字段；`exact` / `keyword` / `semantic` 三个 runnable judge 不读新字段（行为不变），`steps@1`（M2 落地）必读。ADR-0002 patch 把"markdown 由 structured 派生"原则推广到 question。
- M0: math fixture 直接走 ingestion import 路径 seed 10 道题（不绕过）；`scripts/preflight-vision.ts` 30 秒验证 endpoint vision；drift 探针把 math 路径上"`question_id` 兜底 / wenyan-coupled prompt"位点列入 `docs/superpowers/plans/2026-05-22-math-m1-drift-targets.md`。

**Tech Stack:** Drizzle ORM (Postgres) / Next.js App Router / Zod / vitest + testcontainers / @anthropic-ai/claude-agent-sdk via xiaomi/mimo Anthropic-compatible endpoint.

**Spec deltas observed:**
- spec §5 `mathProfile`、`SubjectRegistry` 注册、`JudgeRouteKindSchema` 含 `'steps' / 'multimodal_direct'`、`validateProfile` 完整 — 都**已存在**，本 plan 不重做
- spec §4.2 "image 在派生中丢失" 不准确：`image_refs` 现状走 `question.metadata.prompt_image_refs`（user 已选路 B：promote 为 first-class column）
- spec §3 M-1 scope #3 `question.kind` 增加 `'derivation'` — 用现有 `'calculation'` 即可（`SubjectQuestionKindSchema` 已含）

---

```markdown
## Revision 2026-05-21 — extend to `question` table (M-1)

**Trigger**: math MVP (vision-as-input) requires `question` to carry multimodal
structure (figures, image_refs, structured), not just markdown.

**Rule update**: the principle "markdown is derived from `structured`, not
persisted as the source of truth" extends from `question_block` to `question`.

- `question.structured` (jsonb, nullable) is the source of truth for question
  structure when present.
- `question.prompt_md` and `question.reference_md` are derived (or hand-written
  for non-structured questions); they MUST be regenerable from `structured`
  via `structuredToPromptMarkdown` / `structuredToReferenceMarkdown` when
  `structured` is non-null.
- `question.figures` carries `FigureRef[]` for vision-aware judges (steps@1).
- `question.image_refs` is the simple flat asset_id list for callers that only
  need to fetch images without bbox / attach metadata.
- `question.metadata.prompt_image_refs` is **deprecated** for new writes
  (kept for legacy reader compatibility; remove decision deferred to M3).

**Scope**: `question_block` semantics unchanged (it remains the ingestion-side
structured carrier). What changed: `question` now mirrors the same shape.

**See also**: `docs/superpowers/specs/2026-05-21-math-mvp-vision-design.md` §4.
```

```ts
// scripts/preflight-vision.ts
/**
 * 30-second pre-flight: verify xiaomi/mimo Anthropic-compatible endpoint
 * supports vision input + structured output.
 *
 * Usage:
 *   pnpm tsx scripts/preflight-vision.ts
 *
 * Exit codes:
 *   0 — vision OK, can proceed with M2 vision judge
 *   1 — vision unsupported, must stop and re-decide (downgrade to text-only)
 *   2 — env config missing
 *
 * Output: pass/fail summary + raw model response saved to
 *   docs/preflight/2026-05-21-vision-preflight.json
 */
import { writeFileSync, mkdirSync } from 'node:fs';
import { resolve } from 'node:path';
import Anthropic from '@anthropic-ai/sdk';

const OUT_DIR = resolve(process.cwd(), 'docs/preflight');
const OUT_FILE = resolve(OUT_DIR, `${new Date().toISOString().slice(0, 10)}-vision-preflight.json`);

const apiKey = process.env.ANTHROPIC_API_KEY;
const baseURL = process.env.ANTHROPIC_BASE_URL;
const model = process.env.ANTHROPIC_VISION_MODEL ?? process.env.ANTHROPIC_MODEL;

if (!apiKey || !model) {
  console.error('Missing env: ANTHROPIC_API_KEY and ANTHROPIC_MODEL required');
  console.error('Optional: ANTHROPIC_BASE_URL (xiaomi/mimo endpoint), ANTHROPIC_VISION_MODEL');
  process.exit(2);
}

// A tiny 1x1 red PNG (base64). Enough to test the image path without external fetch.
const TINY_PNG_B64 =
  'iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mP8/5+hHgAHggJ/PchI7wAAAABJRU5ErkJggg==';

async function main() {
  const client = new Anthropic({ apiKey, baseURL });
  const start = Date.now();
  let response: Awaited<ReturnType<typeof client.messages.create>> | null = null;
  let errorMessage: string | null = null;

  try {
    response = await client.messages.create({
      model,
      max_tokens: 256,
      messages: [
        {
          role: 'user',
          content: [
            {
              type: 'image',
              source: { type: 'base64', media_type: 'image/png', data: TINY_PNG_B64 },
            },
            {
              type: 'text',
              text:
                'You are a JSON-only responder. The image is a tiny 1x1 pixel. ' +
                'Respond with exactly this JSON: {"saw_image": true, "color_guess": "<one word>"}',
            },
          ],
        },
      ],
    });
  } catch (err) {
    errorMessage = err instanceof Error ? err.message : String(err);
  }

  const elapsed_ms = Date.now() - start;

  mkdirSync(OUT_DIR, { recursive: true });
  const text =
    response?.content
      .filter((c): c is { type: 'text'; text: string } => c.type === 'text')
      .map((c) => c.text)
      .join('') ?? '';
  let parsedJson: unknown = null;
  try {
    const start_idx = text.indexOf('{');
    const end_idx = text.lastIndexOf('}');
    if (start_idx !== -1 && end_idx !== -1) {
      parsedJson = JSON.parse(text.slice(start_idx, end_idx + 1));
    }
  } catch {
    /* leave parsedJson null */
  }

  const result = {
    timestamp: new Date().toISOString(),
    model,
    base_url: baseURL ?? 'default',
    elapsed_ms,
    error: errorMessage,
    raw_text: text,
    parsed_json: parsedJson,
    vision_supported: errorMessage === null && parsedJson !== null,
  };
  writeFileSync(OUT_FILE, JSON.stringify(result, null, 2));

  console.log(`Pre-flight result: ${result.vision_supported ? 'PASS' : 'FAIL'}`);
  console.log(`Elapsed: ${elapsed_ms}ms`);
  console.log(`Saved: ${OUT_FILE}`);
  if (!result.vision_supported) {
    console.error(`Reason: ${errorMessage ?? 'no parseable JSON returned'}`);
    process.exit(1);
  }
  process.exit(0);
}
```

```ts
// app/api/_/seed/math/route.ts
import { NextResponse } from 'next/server';
import { eq } from 'drizzle-orm';
import { db } from '@/db/client';
import { question, knowledge } from '@/db/schema';
import { createId } from '@paralleldrive/cuid2';
import { loadMathFixtures } from '../../../../../subjects/math/fixtures';

// POST /api/_/seed/math
// Seeds 10 math fixture questions. Idempotent: if a question with the same
// `metadata.fixture_ref` already exists, skip it.
//
// Knowledge nodes: math fixtures use a single placeholder knowledge node
// 'math-seed-root'; M1+ will introduce a real math knowledge graph.
export async function POST(): Promise<NextResponse> {
  const fixtures = loadMathFixtures();
  const now = new Date();

  // Ensure a single placeholder knowledge node exists for math fixtures
  const rootKnowledgeId = 'k-math-seed-root';
  const existingKnowledge = await db
    .select({ id: knowledge.id })
    .from(knowledge)
    .where(eq(knowledge.id, rootKnowledgeId));
  if (existingKnowledge.length === 0) {
    await db.insert(knowledge).values({
      id: rootKnowledgeId,
      label: '数学 seed root',
      domain: 'math',
      created_at: now,
      updated_at: now,
      version: 0,
    });
  }

  const created: string[] = [];
  const skipped: string[] = [];

  for (const item of fixtures) {
    // Idempotency check via metadata.fixture_ref
    const existing = await db
      .select({ id: question.id, metadata: question.metadata })
      .from(question);
    const dup = existing.find(
      (q) => (q.metadata as { fixture_ref?: string } | null)?.fixture_ref === item.ref,
    );
    if (dup) {
      skipped.push(item.ref);
      continue;
    }

    const id = createId();
    await db.insert(question).values({
      id,
      kind: item.kind,
      prompt_md: item.prompt_md,
      reference_md: item.reference_md,
      choices_md: item.choices_md ?? null,
      rubric_json: item.rubric_json ?? null,
      knowledge_ids: [rootKnowledgeId],
      difficulty: item.difficulty,
      source: 'math_fixture',
      variant_depth: 0,
      figures: [],
      image_refs: [],
      structured: null,
      metadata: { fixture_ref: item.ref, knowledge_hint: item.knowledge_hint },
      created_at: now,
      updated_at: now,
      version: 0,
    });
    created.push(id);
  }

  return NextResponse.json({ created, skipped });
}
```
