## Phase goal

Make ADR-0020 executable in the codebase: replace legacy note artifact schema with `body_blocks` / `knowledge_ids`, move correction anchors from `section_id` to `block_id`, and lock the ADR text around the P0 split-id-preserve finding.

## Lane split

### Lane A — Drizzle schema + migration

**Linear**: YUK-91  
**Scope**:
- Modify `src/db/schema.ts` artifact table:
  - drop production use of `knowledge_id`, `child_artifact_ids`, `outline_json`, `sections`;
  - add `body_blocks`, `knowledge_ids`, and `attrs`;
  - add `artifact_block_ref` table + backlink index.
- Add the Drizzle SQL migration and snapshot metadata.
- Add migration smoke coverage for fresh DB and upgrade path.

**Acceptance**:
- `pnpm db:generate` produces the intended migration only.
- `CODEX_FULL_GATE=1 pnpm test:migration` passes.
- No code path still writes old artifact fields after Lane A lands, except intentionally deferred tests noted for P7.

### Lane B — Correction event block anchor rewrite

**Linear**: YUK-91  
**Depends on**: Lane A  
**Scope**:
- Change `CorrectArtifactEvent.payload.section_id` to `block_id` in `src/core/schema/event/known.ts`.
- Update artifact correction projection helpers in `src/server/events/artifact-corrections.ts` and affected tests.
- Update any UI/API call sites that pass section ids for artifact corrections.

**Acceptance**:
- Artifact correction tests assert `block_id` behavior and reject stale `section_id` payloads.
- `CODEX_FULL_GATE=1 pnpm test:unit -- src/core/schema/event/known.test.ts src/server/events/artifact-corrections.test.ts` or the nearest existing targeted tests pass.
- `rg -n "section_id" src app` only finds non-artifact legacy references or documented migration comments.
