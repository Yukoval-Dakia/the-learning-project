# Phase 1a Sub 4A Implementation Plan — Mistake FSRS + review_event + /review

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]` syntax.

**Goal**：错题 FSRS 复习闭环 + 行为日志 + 极简 `/review` UI ship。一周后能跑出真数据回来调 FSRS 参数 / 看 cause 分布 / 评 vision OCR 准确率。

**Architecture**：`ts-fsrs` 库做调度，纯函数 wrapper `workers/src/review/fsrs.ts` 负责 prevState+rating→nextState+dueAt（可单测）；新表 `review_event` append-only 记录每次复习的 before/after 快照；`POST /api/review/submit` 乐观锁更新 `mistake.fsrs_state` + 写 review_event；客户端 `/review` 串行单题 UI 键盘 1/2/3。

**Tech Stack**：Cloudflare Workers + Hono / D1 + Drizzle / `ts-fsrs ^4.x` / React 19 + TanStack Query / zod。

**Spec**：`docs/superpowers/specs/2026-05-10-phase1a-sub4a-design.md`

## 关键决策（lock，详见 spec § 二）

| 决策 | 选择 |
|---|---|
| FSRS 实现 | OSS `ts-fsrs` |
| Rating 映射 | incorrect→again / partial→hard / correct→good（不暴露 Easy） |
| `mistake.fsrs_state` 形态 | 直接对齐 ts-fsrs `Card`（due / stability / difficulty / elapsed_days / scheduled_days / reps / lapses / state / last_review）；JSON 列 |
| `review_event` 表 | append-only（无 update / version / updated_at） |
| Sub 1-3 数据兼容 | 现有 mistake.fsrs_state 全为 null（从未调度过），无迁移成本，直接换 zod 形态 |

Create `workers/src/review/fsrs.ts`:

```ts
import { fsrs, generatorParameters, createEmptyCard, Rating, type Card } from 'ts-fsrs';
import type { FsrsState } from '../../../src/core/schema/business';
import { z } from 'zod';

type FsrsStateData = z.infer<typeof FsrsState>;
type RatingLabel = 'again' | 'hard' | 'good';

const RATING_MAP: Record<RatingLabel, Rating> = {
  again: Rating.Again,
  hard: Rating.Hard,
  good: Rating.Good,
};

const scheduler = fsrs(generatorParameters());

export interface ScheduleResult {
  nextState: FsrsStateData;
  dueAt: Date;
}

/**
 * Pure scheduler. prevState=null means first review (creates empty card).
 * NOW is injected for deterministic tests; real callers pass the wall clock.
 */
export function scheduleReview(
  prevState: FsrsStateData | null,
  rating: RatingLabel,
  now: Date,
): ScheduleResult {
  const card: Card = prevState
    ? cardFromState(prevState)
    : createEmptyCard(now);

  const log = scheduler.next(card, now, RATING_MAP[rating]);
  return {
    nextState: stateFromCard(log.card),
    dueAt: log.card.due,
  };
}

function cardFromState(s: FsrsStateData): Card {
  return {
    due: s.due,
    stability: s.stability,
    difficulty: s.difficulty,
    elapsed_days: s.elapsed_days,
    scheduled_days: s.scheduled_days,
    reps: s.reps,
    lapses: s.lapses,
    state: cardStateFromString(s.state),
    last_review: s.last_review ?? undefined,
  };
}

function stateFromCard(c: Card): FsrsStateData {
  return {
    due: c.due,
    stability: c.stability,
    difficulty: c.difficulty,
    elapsed_days: c.elapsed_days,
    scheduled_days: c.scheduled_days,
    reps: c.reps,
    lapses: c.lapses,
    state: cardStateToString(c.state),
    last_review: c.last_review ?? null,
  };
}

// ts-fsrs uses numeric State enum; we serialize as readable string in DB.
function cardStateFromString(s: 'new' | 'learning' | 'review' | 'relearning'): number {
  // State enum from ts-fsrs: New=0, Learning=1, Review=2, Relearning=3
  return { new: 0, learning: 1, review: 2, relearning: 3 }[s];
}
function cardStateToString(n: number): 'new' | 'learning' | 'review' | 'relearning' {
  return (['new', 'learning', 'review', 'relearning'] as const)[n] ?? 'new';
}
```

Note: ts-fsrs v4 API may differ slightly. If `scheduler.next(card, now, rating)` doesn't exist, check the actual export — common alternatives: `scheduler.repeat(card, now)[rating]` or `scheduler.repeat(card, now).get(rating)`. Adapt to whatever the installed version exposes; surface API mismatches in your status report.

```ts
import { Hono } from 'hono';
import { z } from 'zod';
import { createId } from '@paralleldrive/cuid2';
import { scheduleReview } from '../review/fsrs';
import { FsrsRating } from '../../../src/core/schema/business';
import type { AppEnv } from '../types';

export const review = new Hono<AppEnv>();

review.get('/due', async (c) => {
  const limitRaw = c.req.query('limit');
  const limitParsed = limitRaw ? parseInt(limitRaw, 10) : 20;
  const limit = Math.min(Math.max(isNaN(limitParsed) ? 20 : limitParsed, 1), 50);
  const nowIso = new Date().toISOString();

  const rows = await c.env.DB.prepare(
    `select m.id, m.question_id, m.knowledge_ids, m.cause, m.fsrs_state, m.created_at,
            q.prompt_md, q.reference_md
     from mistake m
     join question q on q.id = m.question_id
     where m.archived_at is null and m.deleted_at is null and m.status = 'active'
       and (m.fsrs_state is null
            or json_extract(m.fsrs_state, '$.due') <= ?)
     order by
       (m.fsrs_state is null) desc,
       json_extract(m.fsrs_state, '$.due') asc,
       m.created_at asc
     limit ?`,
  )
    .bind(nowIso, limit)
    .all<{
      id: string;
      question_id: string;
      knowledge_ids: string;
      cause: string | null;
      fsrs_state: string | null;
      created_at: number;
      prompt_md: string;
      reference_md: string | null;
    }>();

  const out = rows.results.map((r) => ({
    id: r.id,
    question_id: r.question_id,
    prompt_md: r.prompt_md.slice(0, 1000),
    reference_md: r.reference_md ? r.reference_md.slice(0, 1000) : null,
    knowledge_ids: JSON.parse(r.knowledge_ids) as string[],
    cause: r.cause ? JSON.parse(r.cause) : null,
    fsrs_state: r.fsrs_state ? JSON.parse(r.fsrs_state) : null,
    created_at: r.created_at,
  }));
```

```ts
import { FsrsState, FsrsRating } from '../../../src/core/schema/business';

const SubmitBody = z.object({
  mistake_id: z.string().min(1),
  rating: FsrsRating,
  response_md: z.string().nullable().optional(),
  latency_ms: z.number().int().min(0).max(3_600_000).nullable().optional(),
});

review.post('/submit', async (c) => {
  const raw = (await c.req.json().catch(() => null)) as unknown;
  const parsed = SubmitBody.safeParse(raw);
  if (!parsed.success) {
    return c.json(
      {
        error: 'validation_error',
        message: parsed.error.issues.map((i) => `${i.path.join('.')}: ${i.message}`).join('; '),
      },
      400,
    );
  }
  const body = parsed.data;
  const now = new Date();

  // Load mistake
  const row = await c.env.DB.prepare(
    `select id, fsrs_state, version, archived_at, deleted_at from mistake where id = ?`,
  )
    .bind(body.mistake_id)
    .first<{
      id: string;
      fsrs_state: string | null;
      version: number;
      archived_at: number | null;
      deleted_at: number | null;
    }>();

  if (!row || row.archived_at !== null || row.deleted_at !== null) {
    return c.json({ error: 'not_found', message: `mistake ${body.mistake_id} not found` }, 404);
  }

  // CRITICAL: zod-coerce JSON-deserialized strings back to Date.
  // Direct JSON.parse leaves due/last_review as ISO strings, which makes
  // ts-fsrs misbehave (NaN intervals, wrong elapsed_days).
  const prevState = row.fsrs_state ? FsrsState.parse(JSON.parse(row.fsrs_state)) : null;
  const result = scheduleReview(prevState, body.rating, now);

  const dueBefore = prevState ? Math.floor(prevState.due.getTime() / 1000) : null;
  const updateStmt = c.env.DB.prepare(
    `update mistake set fsrs_state = ?, updated_at = ?, version = version + 1
     where id = ? and version = ?`,
  ).bind(
    JSON.stringify(result.nextState),
    Math.floor(now.getTime() / 1000),
    body.mistake_id,
    row.version,
  );
  const insertStmt = c.env.DB.prepare(
    `insert into review_event (
       id, mistake_id, rating, response_md, latency_ms,
       fsrs_state_before, fsrs_state_after, due_at_before, due_at_next, created_at
     ) values (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)`,
  ).bind(
    createId(),
    body.mistake_id,
    body.rating,
    body.response_md ?? null,
    body.latency_ms ?? null,
    prevState ? JSON.stringify(prevState) : null,
    JSON.stringify(result.nextState),
    dueBefore,
    Math.floor(result.dueAt.getTime() / 1000),
    Math.floor(now.getTime() / 1000),
  );

  // Atomic: D1 batch commits both or neither. Hard failure (network/D1 error)
  // → onError 5xx, no partial state. Version mismatch → UPDATE no-op + INSERT
  // commits as audit-only review_event (per spec § 六 invariant).
  const results = await c.env.DB.batch([updateStmt, insertStmt]);
  const updateChanges = (results[0] as { meta?: { changes?: number } }).meta?.changes ?? 0;
  if (updateChanges !== 1) {
    return c.json(
      { error: 'conflict', message: `mistake ${body.mistake_id} was concurrently modified (audit logged)` },
      409,
    );
  }

  return c.json({
    next_due_at: Math.floor(result.dueAt.getTime() / 1000),
    new_state: result.nextState,
  });
});
```
