## Part 1 — 产品方向（发想 13 菜单全集 + 2 早删项）

> 评分 = novelty / AI-native / fit（1-5，来自 repo-blind 研究网）。🪦 抛弃 · ⬇️ 降级 · ♻️ 吸收 · ✅ 保留。

### ⬇️ Reconstruction Engine 重建引擎（5/5/4）
- **是什么**：把「记忆/复习」的基本单位从「一道存好的题」换成「一条推导路径」。传统 spaced repetition（Anki/FSRS）是「到期了把存好的题再考你一遍」；重建引擎反过来——从不复习存货，而是给你一个知识点的**父节点/前置**，让你**当场从地基重新推出它**（「给你乘积法则 + 复合函数，你能不能现推出链式法则」）。掌握 = 「你现在还能不能从地基重建它」，而非「你还记不记得这道题的答案」。理论根=生成效应（generation effect），动机=反叛 Anki「卡片墓地」。
- **为何降级**：novelty-purism 误判——真教研团也用题库，「删 pre-AI 器官」不是 AI-native 判据。
- **去向/复活**：降为教学法脑的一种方法（远期可选，YUK-407）。升核心角色条件=某有自动 ground-truth 的子域实测「重建式」显著优于「调题式」且 owner 想设默认。

### ⬇️ Work Metabolizer 产出反刍器（5/5/5，菜单最高）
- **是什么**：学习语料不再是「你专门坐下来做的题」，而是「你日常真实产出的东西」——代码、PR、commit、设计文档、读书笔记。AI 持续摄入这些产物，自动抽取「你刚刚**隐式**应用或违反了哪个概念」接进 KG（你修了个并发 bug → 它记下「你用到/可能没吃透乐观锁」）。核心卖点=冷启动免费（数据是你工作的副产品，不用额外建卡——99% 真实学习从不被录入正是 Anki 墓地的成因）+ KG 自生长。先例 LOOM 摄入对话、Obsidian 存笔记但不理解也不测验——这个闭环没人做过。
- **为何降级**：泛用/终身向，与「学科教育先行」错配（学科学习是刻意坐下来做的，不是工作副产品，「免费语料」红利打折）。
- **去向/复活**：部分复活为 conjecture 引擎冷启动输入（学科语境下「你的产出」=答案/推理痕迹/笔记）；完整复活=进泛用阶段把 commit/PR 摄入接回。

### 🪦 Long Exposure 慢热长曝光（5/4/4）
- **是什么**：把整个工具当成一场**以「年」为单位的认识论实验**，你自己是被试。进度的定义从「你掌握了多少」变成「你的哪些信念经住了时间的检验、哪些被推翻了」。bi-temporal 数据（你的理解如何随时间演化）从「记忆后端」升格成「实验装置」，让你能回看自己整年的认知轨迹。
- **为何抛弃**：payoff 一年后才兑现，不进早期版。
- **复活**：作可选「看我这一年怎么变」反射层叠加（边际成本低，数据本就在 event 流）；产品稳定运行数月后想纵向回看时捞出。

### 🪦 Calibration Mirror 校准镜（2/3/4）
- **是什么**：它评的**不是你对不对，而是你「知不知道自己知不知道」**（元认知校准）。把你的「自报信心」和「实际正确率」对照，画出 per-domain 的校准曲线（Brier score 轨迹），并把错题分成两类：**confidently-wrong（自信地错——最毒，因为你不会去复查）** vs unsure-wrong（心里没底地错）。
- **为何抛弃**：本质是个 dashboard，AI 在这里只是算指标，非 constitutive（AI-native 仅 2）。
- **复活**：confidence 校准信号已折进 conjecture 的 confidence 字段 + 误解 probe；若想要独立的「校准可视化」面可作 dashboard 捞出，但别当主循环。

### 🪦 Microworld Forge 微世界锻造（4/4/3）
- **是什么**：你**不读解释，而是操作一个当场生成的、可跑的小仿真（sim）**来学——比如学斜抛运动时直接给你一个能拖参数、实时看轨迹的物理沙盒。评估方式从「答题」变成「manipulation-as-assessment」（看你怎么操作 sim 来判断你懂没懂）。源流=Bret Victor「可探索解释」（explorable explanations）。
- **为何抛弃**：不泛化全科（很多学科没有可仿真的对象）+ 即时生成可跑 sim 真贵。
- **复活**：仅在 sim-natural 科目（物理/编程/数学可视化）当一种 content type 捞出，不进通用核。

### 🪦 Adversarial Examiner 对抗考官（5/4/3）
- **是什么**：把学习变成**对抗赛**——你必须主动「击败」一个你最常犯的错误观点。系统拿你的错题库当**对手 roster**，扮演那个持有你 misconception 的对手跟你辩，你要驳倒它。Copilot 在此裂成 opponent（对手）+ referee（裁判）两个角色。
- **为何抛弃**：单独做边际价值低；且「裂人格」违反单人格架构姿态。
- **复活**：作为 Misconception/conjecture probe 的一个**「强度开关」**（同一 misconception 数据，近零边际成本），想要更对抗的体验时打开。

### 🪦 Think-Out-Loud Canvas 思想画布（3/4/3）
- **是什么**：一个**你一边写、它一边 argue back 的笔记本**——你在画布上记录思路/论证，AI 在旁边实时质疑、补漏、追问，像有人陪你边写边辩。artifacts 从「产物」升成主工作区，Copilot 溶进页边的 ambient margin（不抢镜的旁注）。
- **为何抛弃**：它是一个**编辑器 feature，不是一个引擎**（没有独立的学习闭环）。
- **复活**：作为 Work Metabolizer 的输入 UI（你写笔记 → 喂 conjecture 引擎）捞出。

### 🪦 Protégé That Misunderstands 教一个会误解的徒弟（4/5/2）
- **是什么**：你**教一个自信的笨蛋**（AI 扮演一个会犯错、会误解的学生），它的无知和错误**镜像你 KG 的薄弱处**——你教不清楚的地方，正是你自己没吃透的地方。Copilot 角色反转成「被你教的学生」，整段教学 transcript 成为 generation-effect（费曼学习法）的产物。
- **为何抛弃**：单用户 adoption 是口味猜测（fit 仅 2，有人爱有人嫌 AI 装笨很烦）。
- **复活**：Reconstruction/conjecture 体系落地后作可选练习模式 A/B 测；protégé effect（费曼学习法）有真实学习科学支撑，值得日后实测。

### 🪦 Replayable Mind 重放心智（早删）
- **是什么**：一个能**回放你心智成长过程**的叙事化进度面——把你的学习史做成可重播的故事。
- **为何抛弃**：critic 拆穿它本质是进度 dashboard 的豪华版，去掉生成式叙事就退回带时间戳的 stats 页（AI-native 仅 2）；真价值已被 Long Exposure 吸收。
- **复活**：基本不建议；若要，当别的方向的只读反射层。

### 🪦 One Living Document 单一活文档（早删，并入 Think-Out-Loud Canvas）
- **是什么**：你的整个学习沉淀成**一份终身单一的活文档**，AI 当它的 live 维护者（持续帮你重组、更新、链接）。源流=Andy Matuschak 的工作笔记 + 助记媒介理念。
- **为何抛弃**：Matuschak 本人 retrospective 已记录三个杀手问题（surface-pattern trap / authoritarian 输给自治 / conceptual 内容 GPT 失败）+「终身单一巨文档不可导航、无自然停点」基本判了它单人终身尺度的死刑。
- **复活**：其「AI 当 live 维护者」内核已在 Think-Out-Loud Canvas 以 ambient reactive 层存活；「单一巨文档」前提不建议复活。

### ✅ 保留在活计划（不在坟场，列此仅供全景）
- **Grading Twin 判分孪生**（开放题判分 + 记得你标准的演进）→ 教学法/判分轴（Jury），保留。
- **Misconception Engine 误解引擎**（现造诊断 probe 追问你为什么错）→ conjecture 引擎的种子，保留。
- **Curriculum Director 课程总监**（替你排路）→ **规划脑**，保留为 heart，相位 Phase 2。
- **Quitting Engine 弃用引擎**（建模你会不会回来）→ 没单独做成功能，但其关切**升级为贯穿全局的 engagement 过滤器**（tripwire），保留为方法论。
- **Subtraction Engine 减法引擎**（删/合并/降级节点，进度可向下=更整合）→ **复活**为关系脑 async 循环里的一种 proposal type。

## Part 2 — Agent 方法论（被证伪 / 砍的）

> 共同复活条件：出现「cheap 且可信的验证信号」（自动 ground-truth 子域）或多用户规模。

### 🔄 Auto-Quest 自动跨月课程线 — 已复活 YUK-419
> **2026-06-18 复活**为**规划脑的长程层**，重新框定解掉原 kill：**你设目标（保留方向盘）→ 团队拆解+维护 quest 弧 → 规划脑填近期步**；distal 命门靠「近期步 predicted_*/credit 当 quest 体检」缓解。Phase 2+，gated on 规划脑/B1/B3/goal。详见 YUK-419。
- **是什么**：AI 自主规划你未来几个月的整条学习路线、连续替你决定「下一阶段学什么」，像游戏主线任务链自动展开。技术原型=Voyager 式自动课程生成（agent 自己给自己设一串由易到难的目标）。
- **为何砍**：你自己就是课程设计者（强工程师 + 自定目标）；distal feedback（错一个月才知道路线好不好）→ 信任永远赚不回；验收又回到判分。
- **复活**：泛用/非专家学习者场景，或你想交出「学什么」方向盘时。

### 🪦 影子学生模拟（learner simulator / shadow student）
- **是什么**：AI 内部模拟一个「假想的中等学生」，先拿它**预演**某教学策略/题目序列的效果，再决定怎么教你——在真正教你之前先在一个数字替身上试错。源自 Stanford Generative Agents（用 LLM 模拟有记忆、会反应的人）。
- **为何砍**：单用户没有真实学生群体分布去校准这个模拟器，「中等学生」完全是 AI 拍脑袋；它只在有大量真实学生数据时才有校准价值。
- **复活**：多用户后。

### 🔄 元认知反思树（reflection tree）— 已复活 YUK-418
> **2026-06-18 复活**为**关系脑的反思层 = meta-conjectures**（evidence_refs 指向已确认 conjecture）：建在你已确认的 conjecture 之上（非 n=1 原始噪声）+ ≥N 底层 + 待你确认，避开原「统计幻觉」。Phase 1，gated on Phase 0 出货。详见 YUK-418。
- **是什么**：AI 定期从你近期零散表现里**归纳出更高层洞察**（「你近三周第二步老丢分，且都在疲劳晚间」），像一棵不断往上抽象的反思树。直接源自 Generative Agents 的 reflection 机制 + Reflexion（agent 把失败口头总结成经验存进记忆）。
- **为何砍**：你的数据量太小（n=12 这种），「疲劳晚间」是 AI 自己贴的标签——极易统计幻觉，把噪声说成模式。
- **复活**：只在 sleep-time + draft review 内、数据量足够、且经你确认后。**注：关系脑的 conjecture 引擎就是它的「带证据、待你确认」安全版，已采纳。**

### 🪦 技能蒸馏（Voyager skill library / ExpeL）
- **是什么**：把你反复成功的解题套路**蒸馏成可复用的「招式库」**（「这类证明题你先画图，突破率 4/5」），以后遇同类自动调用。源自 Voyager（Minecraft agent 把学会的技能存成可复用代码库）+ ExpeL（agent 从经验提炼规则）。
- **为何砍**：Voyager 能 work 是因为 Minecraft 有**确定性成功信号**（代码跑通=成功）；学习域没有这个信号，「突破率」的分母又回到「算不算突破」=判分；于是蒸馏不可靠。
- **复活**：有自动 verifier 的子域（代码测试通过/数学数值核对）才试。

### 🪦 debate-as-vote（多 agent 投票定真值）
- **是什么**：让多个 AI 互相辩论同一问题，最后**投票/取多数**决定答案或判分对错，期望「群辩比单判更准」。
- **为何砍**：2025 年研究已证伪——多 agent 辩论投票在多数任务上并不比单个强模型更准，是 hype（arXiv:2502.08788）。
- **复活**：不作 vote；但 **debate-as-content**（让 AI 生成正反两面论证给你当学习材料）是安全保留版。

### 🪦 宪法式 rubric 自审（CAI 包装）
- **是什么**：用 Constitutional AI 思路给判分加一层「自我审查」——AI 先按一套书面准则打分，再自己批判修正。
- **为何砍**：CAI 原义是**训练时**用 AI 反馈做 RLAIF（改模型权重），这里其实是**推理时**让模型自我批判一遍，跟 CAI 基本无关、名不副实；相比「直接把准则写进判分 prompt」增量极小。
- **复活**：机制（self-critique）已以「一段准则 + 一次 self-critique pass」留在判分链，只是不贴 CAI 标签。

### ⚠️ swarm 一群互相聊天的 agent（**2026-06-18 重新审视中**）
- **是什么**：把单一 copilot 炸成一群各司其职、互相通信协作的 agent（导师/考官/教练各一个进程），像一个在群聊里干活的团队。
- **曾经为何砍**：MAST taxonomy 记录的多 agent 失败模式（误传、错误传播）；单用户下是 complexity for its own sake。
- **重新审视（owner 2026-06-18 要求，workflow `woq3hjqap` 进行中）**：发现关键区分——之前否的是**「为协调而 swarm」**（多 agent 协同产出**单一**结果，脆弱）；但**「为分歧而 panel」**（多个真独立异构视角各执一词、分歧本身是产物 + 审议过程即产物）是另一回事，可能正是「团队感」的来源且 robust。Jury 异构再归因已是此模式。**最终形态待 workflow 回 + owner 拍。**

## Part 3 — 架构 / 框架立场（被推翻的）

### 🪦 bi-temporal on KG（valid_at / invalid_at on edges）
- **是什么**：给知识图谱的边加两条时间轴——valid-time（这条事实在世界里何时为真）+ transaction-time（系统何时记下），支持 as-of 查询（「2026 年 3 月那天这棵知识树长什么样」）。源自 Graphiti（服务多 agent 知识图谱的「事实时效」）。
- **为何推翻**：KG 存的是 timeless 认知结构（「A 是 B 的前置」不会过期）；「不再为真」是 epistemic 纠错（CorrectionKind）非 valid-time；单用户无 as-of 消费者（ADR-0034）；时变 learner-state 走 event 流 + 衰减；valid-time 唯一合法落点是 mem0 软画像。
- **复活**：开 superseding ADR + 出现真实「审计某历史时刻结构态」消费者（多用户后可能）。

### 🪦「删题库 / AI-native = 避开 pre-AI 形态」（novelty-purism）
- **是什么**：把「不像 Anki/题库/spacing」当作 AI 原生的判据——认为要惊艳就得删掉所有 pre-AI 形态的东西。
- **为何推翻**：owner 第二次纠偏——真教研团也用题库、也调题、也安排练习和考试；题库是工具不是原罪。
- **复活**：不复活（错误前提）。

### 🪦「Anki 导出测试」当 constitutive 判据
- **是什么**：用「产物能不能无损导成一副 Anki deck」当「你造的是不是 AI 原生新东西」的判据（能导出=你还是 Anki）。
- **为何推翻**：真团队的题库也能导出，判据本身坏。
- **复活**：不复活；换成正确标尺「是否给一个人提供以前不可能的专属团队级关注」。

### 🪦「这只是 AI 加法、功能够了」原始前提
- **是什么**：认为现有 copilot + 题库 + 题流 + artifacts + 知识点已经够了，只需继续往上叠 AI 功能。
- **为何推翻**：整个 rethink 的起点就是推翻它——拿掉 AI 只变慢变贵不坍塌 = AI 加法，不是 constitutive。
- **复活**：不复活。

## 两条 framing 教训（owner 两次纠偏的元收获）
1. **历史陷阱**：grounding 会把 rethink 拽回**增量主义**——「你已造好一半」可能是洞察也可能是合理化，从 grounding 内部分不出。
2. **新奇洁癖**：矫枉过正会冲进「**为不一样而否定旧工具**」。
3. **守则**：判据不是「像不像旧工具」，而是「是否给一个人提供以前不可能的专属团队级关注」+「第 90 天更想回来还是关掉」。codebase 只答「先做哪个/多贵」，永不答「做不做/敢不敢」。
