#### [ADR-0032 D4] `propose_knowledge_edge` 缺少 archive 判别式

- **声明** (ADR-0032 D4, E-1): "`propose_knowledge_edge` 现为 create-only，无删边路径 → 改为判别式 `propose_create | propose_archive`，镜像节点侧 omnibus"
- **代码**: `src/server/ai/tools/proposal-tools.ts:493` 的 `propose_knowledge_edge` 工具只实现 create 路径；schema (`knowledge_edge.archived_at`) 和 read filter (`isNull(knowledge_edge.archived_at)`, knowledge-readers.ts:84) 已就绪，但 propose 侧无 archive 分支。
- **问题**: Agent 无法通过工具删除（archive）知识 mesh 边，形成单向写死；知识图谱维护能力不完整。
- **建议**: 在 `propose_knowledge_edge` 输入 schema 加 `mutation: 'propose_create' | 'propose_archive'` 判别式，archive 分支写 `knowledge_edge.archived_at = now()`。

  - title: `Drift: ADR-0032 D4 — propose_knowledge_edge missing archive discriminant`
  - labels: `drift`
