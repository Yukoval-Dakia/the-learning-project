### ⚠️ Documented-only

#### [ADR-0046 §4] A13 proper-scoring kernel — Rust-first 首试验未落地

- **声明**: ADR-0046 §4 明确指定"A13 proper-scoring kernel = **首个 Rust-first 试点**（净新、边界干净、不碰 live θ̂）"；新数值 kernel 一律 Rust-first、无 TS oracle。
- **代码**: `src/server/conjectures/scoring.ts:1-6` 是 TS PLACEHOLDER stub，自注"ADR-0046: proper-scoring = Rust-first single-source-of-truth. This TS is a PLACEHOLDER stub — bit-exact replacement lands with the Rust calibration kernel (crates/calibration-native)"；但 `crates/calibration-native/src/lib.rs` 目前只含 mulberry32 / forwardAuc / bootstrap（YUK-493 Phase -1 搬运）——没有 Brier/log-loss/skill-score 函数。
- **冲突**: ADR 指定的首个 Rust-first 试点（proper-scoring）尚未落到 Rust crate；TS 占位符在生产中运行，单点评分计算实际仍在 TS。
- **建议**: 在 `crates/calibration-native` 实现 `score_prediction` → napi binding，将 `scoring.ts` 切为 binding 调用者，删除 TS 算法体。此为 ADR-0046 §4 要求的首个 Rust-first 交付，完成后删 TS 实现。
- Linear issue:
  - title: `Drift: ADR-0046 A13 proper-scoring kernel not ported to Rust`
  - labels: `drift`

#### [docs/design/2026-06-14-product-rethink-decisions-ledger.md §B1] `knowledge_mastery` view `evidence<3→0.5` placeholder 未清除

- **声明**: B1 决策明确"删 `knowledge_mastery` view 的 `evidence<3→0.5` 占位，换 PFA logistic"；`mastery_state` 表（ADR-0035）取代 view 成为 p(L) 诊断维 SoT（YUK-420 起 view 消费端已切）。
