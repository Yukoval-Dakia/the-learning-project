#### [Mem0 ANTHROPIC_BASE_URL Env Mutation] Process-global env dance not documented in ADR-0017 errata

- **声明**: ADR-0017 erratum (2026-05-27) acknowledges "LLM path remains Anthropic provider compatible, routed through `ANTHROPIC_BASE_URL` plus `XIAOMI_API_KEY`" in one sentence. No detail on implementation contract.
- **代码**: `src/server/memory/client.ts:110–170` — `withXiaomiBaseUrl()` temporarily mutates `process.env.ANTHROPIC_BASE_URL` for the synchronous `Memory()` constructor call; a Mutex (`YUK-232 [SEC-2]`) serialises callers to prevent interleaved global mutation. Comments note this is safe ONLY because the ctor chain is synchronous.
- **冲突**: The Mutex + synchronous-window invariant is a load-bearing safety constraint. If the invariant is violated (async code added inside the window, or mem0ai's Memory ctor becomes async in a future version), the env mutation becomes a race. ADR-0017 errata don't capture this, so maintainers lack the signal to react when upgrading mem0ai.
- **建议**: Add an errata entry to ADR-0017 documenting: (a) `withXiaomiBaseUrl` is required because mem0ai `AnthropicLLM` ignores `config.baseURL`; (b) the mutex + synchronous-window invariant; (c) re-check trigger: "Revisit when installed mem0ai gains baseURL forwarding for Anthropic provider — then drop env dance."
- Linear issue（closeout gate）:
  - title: `Drift: ADR-0017 errata ↔ src/server/memory/client.ts (env-mutation contract undocumented)`
  - labels: `drift`
