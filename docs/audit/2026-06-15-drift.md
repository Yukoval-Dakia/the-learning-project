#### [ADR-0038 §D5] QuizVerify `'error'` channel missing from `QuizVerificationResult.overall`

- **声明**: ADR-0038 D5（2026-06-14）：Wave 0「独立无依赖，先做」—— `QuizVerificationResult.overall` 扩 `'error'` 通道，区分「transport/parse/DB 瞬态故障（verdict 未产出）」与「真实 fail verdict（验了判不合格）」，对齐 `quiz_verify.ts:189-192` pg-boss redelivery 语义。
- **代码**: `src/core/schema/quiz_gen.ts:L271`，`overall: z.enum(['pass', 'needs_review', 'fail'])`；缺 `'error'` 项。
- **冲突**: Wave 0「立即可起跑」项未实现；下游 `quiz_verify.ts:189-192` 的 `outcome='error'` 隐式区分在 result schema 层仍不可见，调用方无法区分重试态与弃用态。
- **建议**: `quiz_gen.ts:271` 改为 `z.enum(['pass', 'needs_review', 'fail', 'error'])`，更新 `quiz_verify.ts` rollup 逻辑把 transport/parse 异常路 map 到 `'error'`，单测覆盖两种 redelivery 语义分支。
- Linear issue（去重后创建，label `drift`）：
  - title: `Drift: ADR-0038 D5 ↔ quiz_gen.ts missing error channel`

#### [ADR-0033 §D4] Sandbox iframe renderer for `type='interactive'` artifacts not built

- **声明**: ADR-0033 D4（2026-06-09）：「新建沙盒 iframe 渲染层（srcdoc + sandbox 去 same-origin + CSP 禁网）——安全 greenfield」；D6：copilot `author_artifact(type='interactive')` 写工具。后续 re-grounded plan（2026-06-13）明记「SPA persistent render path not wired (write-only gap)」。
- **代码**: `src/core/schema/business.ts:L125`（`ArtifactType='interactive'` 存在）+ `src/server/ai/tools/author-artifact.ts`（后端写工具存在）；`web/` 内 `srcdoc`/`sandbox`/`InteractiveArtifact` 零命中——SPA 渲染层不存在。
