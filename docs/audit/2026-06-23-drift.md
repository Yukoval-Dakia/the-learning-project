#### [ADR-0039 §6a] ambient_context 仍是 chat.ts 私有入参，未升格 Postgres 表

- **声明**: "把 `ambient_context` 从 `chat.ts` 私有入参升格为正式表，存当前面 / focused_entity / 上一轮练习结果 / **刚 dismiss 哪条**"。「刚 dismiss 哪条」进工作记忆 → D14 的 B 档 dismiss 回流闭环。
- **代码**: `src/capabilities/copilot/server/chat.ts:168` — `ambient_context: z.record(...)` 仍是请求 schema 的可选字段，无对应 Postgres 表（`Grep src/db/schema.ts ambient_context` 零命中）。
- **缺口**: dismiss 回流闭环物理不存在——编排者不记得上一轮 dismiss 了哪条，B 档 dismiss 信号无法跨请求持久化，「不重复同类提议」红线无法执行。
- **建议**: 在 schema.ts 加 `copilot_session_context` 表（面/focused_entity/recent_dismiss/上次练习结果），`chat.ts` handler 写入 dismiss 事件时同步更新，构造 prompt 时读取；先做 dismiss 字段，其余字段可分步。
- **Linear**: title `Drift: ADR-0039 §6a ↔ ambient_context 未升格 Postgres 表 (dismiss 回流断)`, labels `drift`

#### [ADR-0039 §4] defer / archive / judge_retraction 仍留在 inbox 裁决面

- **声明**: "defer / archive / judge_retraction 移出裁决面（落 C 档）。它们是状态 / 旁观动作不是『需人审的结构写入』，本不该占裁决 lane。"
- **代码**: `src/core/schema/proposal.ts:16-20` — `'defer'`, `'archive'`, `'judge_retraction'` 仍在 `aiProposalKinds` 数组；`acceptSupportedProposalKinds` 注释（L74-76）明确「这三 kind 走 default 分支抛 unsupported_proposal_kind 400」，UI ProposalCard 据此门控 Accept CTA 不展示。YUK-44 未解。
- **缺口**: 三 kind 出现在 inbox 但无法 accept，用户只能 dismiss——产生「只有 dismiss 按钮的空白提议卡片」UX 负债；且它们占据 inbox 真正裁决 lane 的信噪比。
- **建议**: 将三 kind 从 `aiProposalKinds` 移出，重路由到：`defer → snooze 控件` / `archive → 直接软归档调用` / `judge_retraction → agent-notes 旁观只读`；`inbox-meta.unit.test.ts` 已钉住集合不变量，迁移时同步更新测试。
- **Linear**: title `Drift: ADR-0039 §4 ↔ defer/archive/judge_retraction 仍在 inbox 裁决面`, labels `drift`
