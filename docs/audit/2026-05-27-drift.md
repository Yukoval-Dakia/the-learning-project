#### [ADR-0004 / ADR-0017] `TaskMiddleware { beforeRun, afterRun }` hook in `src/server/ai/runner.ts` — 未在任何 ADR 中声明

- **声明**: ADR-0004 描述 runner 使用 Claude Agent SDK query loop；ADR-0017 描述 memory 层由 `src/server/memory/client.ts` → Mem0 实现。两份 ADR 均无涉及 runner 本身的 hook 机制。
- **代码**: `src/server/ai/runner.ts:69-86` 定义 `TaskMiddleware: { beforeRun?(kind, input, ctx): Promise<unknown>; afterRun?(kind, result, ctx): Promise<void> }` 并在 `runTask` / `runAgentTask` / `streamTask` 三条路径上全量织入；注释说 "Memory module decorates input ahead of the model call and observes output after"。目前无生产调用方。
- **冲突**: Runner 引入了 memory 层集成的 hook 接口，但 ADR-0017 仅描述 `src/server/memory/client.ts` 作为写入端入口，未说明 memory 读取（recall context 注入到 task input）通过 runner hook 实现。`src/server/memory/` 目录不存在，hook 合约与 ADR-0017 的 Mem0 架构描述之间存在集成路径未对齐。
- **建议**: 在 ADR-0017 §"Layer integration" 补一节，说明 runner `beforeRun` hook 是 memory recall 上下文注入点，`afterRun` hook 是 memory write-back 触发点；或在 ADR-0004 §"共享薄 harness" 中追认 `TaskMiddleware` 接口。
- **Linear issue**: title `Drift: ADR-0017 ↔ src/server/ai/runner.ts TaskMiddleware`，labels `drift`
