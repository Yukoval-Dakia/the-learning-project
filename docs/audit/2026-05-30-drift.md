## D-ii — Profile validator 7-invariant re-audit

**Roadmap 声明的 7-invariant 目标**（`docs/planning/v0.4-complete-form-roadmap.md` line 235，"完全体目标"）：

> profile validator 检 7 invariant（pipeline schema 兼容 / fallback 同 family / renderer 装了 / prompt 不引用缺失 section 等）

注：roadmap 仅显式点名 4 项 + "等"；line 240 自陈 "profile validator 是否真检了 round 6 codex 列的 7 invariant **待 audit**"。本节即该 audit。

**`src/core/capability/validate-profile.ts` 实际检查的内容**：

| # | 实际检查项 | 代码位置 | 对应 roadmap 目标 |
|---|---|---|---|
| 1 | `SubjectProfileSchema` Zod shape | L142-149 | 基础 schema 合法 |
| 2 | `version` 非空字符串 | L152-154 | — |
| 3 | `causeCategories` 存在 / 格式 / id 唯一 | L24-47, L156 | — |
| 4 | `judgeCapabilities` 是 string / 唯一 / 在 registry 注册（deprecated→warn）+ `judgePolicy.preferredRoutes` registry-backed route 必须被声明 | L49-94, L157 | 部分对应 "renderer/judge 装了" |
| 5 | `renderConfig` 合法（`RenderConfig.safeParse`） | L159-162 | ✅ "renderer 装了" |
| 6 | `schedulingHints` 合法 + `default_policy` 解析到已注册 scheduler 且该 scheduler `supports_activity_kinds` 含 `'question'` | L104-134, L164-171 | scheduler policy 注册 |

**结论：DRIFT — validator 实检 ~4 类语义不变式，roadmap 目标列 7 个。覆盖差距 3 项：**

1. ⚠️ **"prompt 不引用缺失 section"** — **完全未检**。validator 不校验 profile 的 prompt fragment / section 模板是否引用了 renderConfig 里不存在的 section。这是 roadmap 显式点名的目标，是真实 gap。
2. ⚠️ **"fallback 同 family"** — **未检**。validator 校验 `preferredRoutes` 里 registry-backed route 已被声明（L87-93），但不校验 judge fallback chain 是否在同一 capability family（roadmap 语义）。
3. ⚠️ **"pipeline schema 兼容"** — **部分 / 未交叉校验**。CapabilityManifest 声明了 `inputSchema`/`outputSchema`（registry 层），但 validator 不交叉核对 profile 选的 capability 的 pipeline schema 与 profile 期望是否兼容。

**判断**：这是 roadmap 完全体目标 vs 当前实装的差距，**不是 ADR↔code 矛盾**（ADR-0014 §C 没硬性要求 7 个全实装在 validate-profile.ts 这一个函数里；部分由 `SubjectRegistry.register()` 启动校验 + `pnpm audit:profile` 分担）。所有 3 个 profile 当前 `pnpm audit:profile` 全绿（profiles: 3 / invalid: 0 / warnings: 0），说明现有 4 类检查对现有 profile 足够。差距在"未来 onboard 第 4+ 学科时，prompt-section reference / fallback-family / pipeline-schema 兼容性 不会被自动挡住"。

**Actionable flag（交 coordinator 进 Linear）**：profile validator 7-invariant 覆盖差距 —— 补 "prompt 不引用缺失 section"（最高价值，roadmap 显式点名 + 直接影响渲染正确性）+ "fallback 同 family" + "pipeline schema 兼容" 三项校验，或在 roadmap 把目标从 "7 invariant" 下修到实际的 ~4 项 + 注明其余由 registry/audit 分担。本 doc-sweep 按约定 **不改 validator 代码**。
