#### [Plan 2026-05-18] `learning_record` + `memory_brief_note` — 一等领域表无 ADR

- **声明**: `docs/superpowers/plans/2026-05-18-learning-data-loop.md` 引入 `learning_record`（user-visible activity context）和 `memory_brief_note`（Dreaming 维护的学习记忆摘要），定位为"第三类主要持久化实体"（与 `event` + `learning_session` 并列）。
- **代码**: `src/db/schema.ts:L210`（`learning_record` pgTable）、`src/db/schema.ts:L244`（`memory_brief_note` pgTable）；写路径在 `src/server/records/queries.ts:L74`（insert）、`L149`（update）；Migration `drizzle/0007_learning_record_loop.sql`。
- **问题**: CLAUDE.md 要求架构决策记录在 `docs/adr/ADR-NNNN.md`。ADR-0006 v2 覆盖 `event`，ADR-0008 覆盖 `learning_session`，但 `learning_record` 的核心决策（为什么不用 event 直接表达 mistake/worked_example、为什么不用 learning_item）没有 ADR 追认。未来新人或 Agent 读 ADR 时无法了解这两张表的设计原则与边界。
- **建议**: 新建 ADR-0015，覆盖 `learning_record` 的语义边界（与 `event` 的关系：event 是不可变 action log，learning_record 是用户可见的 activity context tag）、`memory_brief_note` 的 Dreaming 写路径约束、以及两者的单一所有者原则（对齐 ADR-0005）。

#### [ADR-0003/0014 §profile-migration] `registry.ts` 保留死代码 `systemPrompt` 字符串

- **声明**: ADR-0003（§2026-05-17 修订）+ `docs/superpowers/plans/2026-05-19-profile-blind-ai-tasks-closeout.md`：所有 AI task 的 system prompt 应通过 `getTaskSystemPrompt(kind, subjectProfile)`（`src/ai/task-prompts.ts`）动态渲染，以支持多学科。runner 在 `src/server/ai/runner.ts:L223` 始终调用 `getTaskSystemPrompt`，从不直接读 `tasks[kind].systemPrompt`（除 `default` 分支兜底）。
- **代码**: `src/ai/registry.ts:L180`—`TeachingTurnTask.systemPrompt` 仍为 `"你是文言文学习教练...《师说》《伶官传序》..."`；`L152`—`NoteGenerateTask.systemPrompt` 含 `"文言文示例首选经典原文"`；`L165`—`VariantGenTask.systemPrompt` 同。这 9 个 task 的静态 prompt 已被 `task-prompts.ts` 覆盖，registry 里的字符串是死代码。
- **问题**: 没有任何 ADR 或 plan 标记 `registry.ts` 的 `systemPrompt` 字段为"legacy / deprecated"。AI Agent 或新人读 registry 时会拿到错误的 prompt 文档（wenyan-only），与实际运行时行为不符。若未来有人直接引用 `tasks[kind].systemPrompt` 绕开 `getTaskSystemPrompt`，会悄无声息地退化为文言文专有 prompt。
- **建议**: 选一：(a) 在 `registry.ts` 每个已迁移 task 的 `systemPrompt` 字段加注 `// deprecated: runtime uses task-prompts.ts getTaskSystemPrompt()`，并在 `getTaskSystemPrompt` default 分支加 warn；(b) 将 `systemPrompt` 字段从 `TaskDef` 类型移除，改为 optional，只保留 `ReviewIntentTask` 等未迁移的 task 的字段。
- `gh issue create`:
  ```bash
  gh issue create \
    --title "Drift: registry.ts 死代码 systemPrompt (文言文硬编码) 与 task-prompts.ts 不同步" \
    --label "drift,automated" \
    --body "docs/audit/2026-05-20-drift.md §[ADR-0003/0014] 条目。\n\n\`src/ai/registry.ts\` 里 TeachingTurnTask（L180）、NoteGenerateTask（L152）、VariantGenTask（L165）等 9 个 task 的 \`systemPrompt\` 是死代码——runner 始终走 \`getTaskSystemPrompt()\`。wenyan 硬编码文本仍在 registry，误导读者。建议 deprecate 标注或移除字段。"
  ```
