### ⚠️ Documented-only

#### [ADR-0004 §"审计" + ADR-0006 v2 §event schema] `ai_task_runs` 表未落地

- **声明**:
  - ADR-0004 §"第一类：Backend Purpose Agent" 审计栏: `ai_task_runs + ai_tool_calls + ai_cost_ledger`
  - ADR-0006 v2 event schema（line 56）: `task_run_id: text NULLABLE  → ai_task_runs    -- AI 事件关联 ai_task_runs`
  - `docs/architecture.md:569`: `task_run_id → ai_task_runs`
- **代码**: `src/db/schema.ts` 无 `ai_task_runs` 表；`event.task_run_id` 是无 FK 的 free-form text；`src/server/ai/runner.ts:309` 在 `RunTaskResult` 中返回 `task_run_id`（`createId()` 生成），但**不插入任何 task run 汇总行**。
- **冲突**: 三处文档均声明 `ai_task_runs` 是 AI 任务审计的必要表；该表缺失意味着 `task_run_id` 成悬空引用——无法通过 `task_run_id` 查询 "task run X 的完整上下文、参数、模型配置"。`tool_call_log` + `cost_ledger` 仍以 `task_run_id` 分片写入（可查单步），但缺少汇总行。
- **建议**: 选项一——实装 `ai_task_runs` 表（id, kind, provider, model, input_hash, started_at, finished_at, finish_reason, usage_json）并在 runner 中 INSERT；选项二——若决策已改为"不需要汇总行"，则更新 ADR-0004 / ADR-0006 v2 / architecture.md，把 `task_run_id` 注释改为"自生成 CUID，关联 tool_call_log / cost_ledger，无独立 task run 表"。
- `gh issue create`:
  ```bash
  gh issue create \
    --title "Drift: ai_task_runs audit table documented but not implemented" \
    --label "documentation,drift" \
    --body "ADR-0004, ADR-0006 v2, and architecture.md all reference an 'ai_task_runs' table as the backing store for event.task_run_id. The table does not exist in schema.ts. task_run_id in the event table is a dangling free-form text with no authoritative row. Either implement the table (runner INSERT on each task run) or update all three docs to reflect the actual design (no summary row, only tool_call_log + cost_ledger per step)."
  ```
