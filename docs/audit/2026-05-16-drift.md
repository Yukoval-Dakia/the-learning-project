#### [ADR-0002 §持久化] extracted_prompt_md 仍在 import route 持久化

- **声明**: ADR-0002 §"结构化存储": "extracted_prompt_md 不持久化，所有 markdown 视图由 structured 现场派生（structuredToPromptMarkdown()），杜绝双源不一致。"
- **代码**: `app/api/ingestion/[id]/import/route.ts:217`：`extracted_prompt_md: block.final_prompt_md`——client 传来的 markdown 写入 DB。Route 自身注释（L14-17）承认此为"混合 schema"向前兼容行为：pre-Sub-0c blocks 没有 structured，只有 extracted_prompt_md；post-Sub-0c blocks 有 structured，extracted_prompt_md 为 null，但 import 路径依然覆写它。
- **漂移**: 对 post-Sub-0c blocks，import 步骤将 `final_prompt_md`（用户在 review UI 确认的文本）写入 `extracted_prompt_md`，产生 ADR-0002 明确禁止的"双源"（structured + extracted_prompt_md 同时非 null）。Phase 1c.1 Step 4/6 import 路由重写时应同步清理。
- **建议**: Phase 1c.1 Step 4/6 重写 import route 时：(a) 对 post-Sub-0c blocks，不再写 `extracted_prompt_md`（置 null 或不更新）；(b) 前端 review UI 使用 `structuredToPromptMarkdown(block.structured)` 派生展示用 markdown，不依赖 extracted_prompt_md；(c) pre-Sub-0c blocks 的 backward compat 需要一次性迁移（parsed structured 回填或标记为 legacy-only）。
