### ❌ Contradicted

#### [ADR-0015 §1] `learning_record.kind` contract source — ActivityKind vs LearningRecordKind

- **声明**: ADR-0015 §1 L47 `kind ∈ ActivityKind enum（attempt / read / teach_turn / ...）` + L96 "ADR-0014 の ActivityKind / CapabilityRef registry は learning_record `kind` / `payload` の契約源"。两处明确把 `src/core/schema/activity.ts` 里的 `ActivityKind` 指为 `learning_record.kind` 列的值域来源。
- **代码**: `src/core/schema/business.ts:64` 定义 `LearningRecordKind = z.enum(['mistake', 'worked_example', 'open_question', 'insight', 'reflection', 'observation', 'resource_note'])`，该 enum 被 `src/server/records/types.ts:6` 导入并用作 `CreateLearningRecordInput.kind` 的类型。`activity.ts` 里的 `ActivityKind`（`'question' | 'question_part' | 'record' | 'recall_prompt' | 'practice_log' | 'project_milestone' | 'open_inquiry'`）与 `LearningRecordKind` 值域完全不重叠。
- **冲突**: ADR-0015 指向 ActivityKind，但代码实际用的是 LearningRecordKind。两个 enum 语义不同：ActivityKind 表示"被学习的材料种类"（ADR-0014 §1），LearningRecordKind 表示"记录内容种类"（错题 / 例题 / 洞察…）。ADR-0015 引用的示例值 'attempt / read / teach_turn' 既不在 ActivityKind 也不在 LearningRecordKind——说明 ADR 写定时 enum 未最终化，后续独立演化未回写 ADR。
- **建议**: 修正 ADR-0015 §1 L47 引用从 `ActivityKind` 改为 `LearningRecordKind`（`src/core/schema/business.ts:64`），删除 L96 "ADR-0014 の ActivityKind は契約源" 表述（该说法不适用于 learning_record.kind 列），明确 `learning_record.kind` 与 `ActivityRef.kind` 是语义独立的两个 enum。无需改代码——代码侧 LearningRecordKind 行为正确，只是 ADR 没追上。
- Linear issue（closeout gate）：
  - title: `Drift: ADR-0015 §1 ↔ src/core/schema/business.ts:64 — LearningRecordKind vs ActivityKind mismatch`
  - labels: `drift`
