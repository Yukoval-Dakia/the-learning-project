#### [ADR-0039 §6a] `ambient_context` 未升格为正式 Postgres 表

- **声明**: "把 `ambient_context` 从 `chat.ts` 私有入参升格为正式表，存当前面 / focused_entity / 上一轮练习结果 / **刚 dismiss 哪条**"（ADR-0039 决定 6a）
- **代码**: `src/capabilities/copilot/server/chat.ts:168` 中 `ambient_context` 仍是 Zod per-run 字段（随 HTTP 请求传入，进程内不留存）；`src/db/schema.ts` 无 session_work_memory 或同名表——「刚 dismiss 哪条」无法跨 turn 持久化
- **冲突**: ADR-0039 要让 B 档 dismiss 回流闭环（"编排者下次不重复同类提议"），其前提是 dismiss 记录能在 Postgres 跨对话读；当前 ambient_context 仅活在单次请求，dismiss 回流死环未闭
- **建议**: 新建 session_work_memory（或同名）Postgres 表；字段含 route/focused_entity/last_practice_outcome/**dismissed_kind[]**；updateAmbientContext() 写入，copilot chat 启动时读取替代 per-run 入参
- Linear issue（closeout gate）：
  - title: `Drift: ADR-0039 §6a ↔ chat.ts — session working memory 未建表`

#### [ADR-0039 §1 + decision risks] CORE_TOOLS bootstrap latch 仍存活，4 后台 job 未并入 D14

- **声明**: "单编排者 D14 = 前台 Copilot + 后台 4 job 合为同一编排者的不同召唤姿势"（ADR-0039 决定 1）；"须先让 copilotTools 贡献制真生效（退役 CORE_TOOLS latch）再谈 job 收编，否则「合为同一 D14」只是叙事不是真相源"（ADR-0039 代价/风险）
