#### [ADR-0017 §"Write triggers" #2] Chat-derived memory write path not implemented

- **声明**: ADR-0017 §"Write triggers (three paths)" #2 — "Chat-derived — Copilot conversation turns can produce user-preference facts ('user prefers analogies', 'user wants reasoning chain shown'). Per turn, the orchestrator decides whether to call `mem0.add(chat_message, scope='meta:orchestrator_self')`. Mem0's internal extraction filters non-signal turns."
- **代码**:
  - `app/api/copilot/chat/route.ts` — 无 mem0 / memory 调用
  - `src/server/ai/tools/allowlists.ts` — `search_memory_facts` 在 copilot allowlist，但**无对应 write 工具**
  - `src/server/memory/client.ts:addEventMemory` — 仅被 `triggers.ts:138`（event ingest outbox worker）调用；chat 路径不触达
  - Path #1（event-ingest outbox，ADR-0021）✅ 已实现；Path #3（cron daily sweep，`handlers.ts:398-400`）✅ 已实现；仅 #2 缺失
- **冲突**: ADR-0017 声明三条写入路径，当前仅 paths #1、#3 落地；chat-derived memory write 既无 tool 注册也无直接调用路径，Copilot 不能在对话中向 Mem0 写入 `meta:orchestrator_self` 偏好事实。
- **建议**: 在 `src/server/ai/tools/` 新增 `write_memory_fact` DomainTool（effect='write', costClass='cheap_llm'），写 `src/server/memory/client.ts::addChatMemory`（scope='meta:orchestrator_self'），加入 `DOMAIN_TOOL_ALLOWLISTS.copilot`（仅 copilot，D6 治理不变），并配 `allowlists.test.ts` 守护，防止 review_plan / evaluator 等路径误授权。
