### ❌ Contradicted (优先)

#### [ADR-0038 §决定3] Variant verify-then-promote 翻转未落地
- **声明**: "VariantGen 产出先落 `draft`/pending，过统一 verify 契约 `overall='pass'` 后才 promote"（翻转此前 accept-first 模式）
- **代码**: `src/capabilities/practice/server/proposal-appliers.ts:104-230` `acceptVariantQuestionProposal` accept 时即写 `draft_status='active'`，`VariantVerifyTask` 之后才异步跑
- **冲突**: 题目在统一 verify 契约跑之前已 pool-visible，ADR 要反转的 accept-first 信任反转没做
- **建议**: 改材料化顺序为 verify pass 后才 promote 到 active，或在 ADR 加 phase-deferred 注记
- Linear issue（label `drift`）：
  - title: `Drift: ADR-0038 variant verify-then-promote not implemented ↔ proposal-appliers.ts`

#### [ADR-0046 §决定1/3] Rust 数值引擎单一源尚未进生产
- **声明**: "θ̂ / PFA / calibration 一份 Rust 实现，napi 喂服务端…TS 不再重写" + "binary ship 进生产镜像"
- **代码**: `src/core/theta.ts:23` / `src/core/pfa.ts:42` 仍活跑 TS sigmoid（`POLY_SIGMOID_ENABLED=false`）；`Dockerfile` 无 cargo/napi 构建步骤
- **冲突**: ADR 描述的"反转已完成"与生产现状不符——TS 仍是 live oracle，双实现漂移风险未解
- **建议**: 若按 ADR 自身"触到才迁"的渐进计划走，属预期未完成，需在 ADR 补一句避免"已完成"措辞误导；否则排期迁移
- Linear issue（label `drift`）：
  - title: `Drift: ADR-0046 Rust engine not yet live in prod ↔ theta.ts/pfa.ts`
