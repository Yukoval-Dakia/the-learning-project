#### [ADR-0039 §1 note + ADR-0037 §5] `review_plan` job claimed retired but still live and enqueued
- **声明**: ADR-0037 §5 "review_plan job 随即退役/全删，不再独立注册"; ADR-0039 §1 note explicitly asserts "review_plan job 本身已按 ADR-0037 退役（不再独立注册/另开 paper channel）——别照字面去维护一个 ADR-0037 要消除的 review_plan surface"
- **代码**: `src/capabilities/practice/manifest.ts:220-222` still registers `review_plan` as an active pg-boss job; `src/capabilities/agency/jobs/coach_daily.ts:485` still calls `await boss.send('review_plan', payload)`; `src/capabilities/practice/jobs/review_plan.ts` is a full active handler running `ReviewPlanTask` over 4 narrow DomainTools
- **冲突**: ADR-0039 makes a positive factual claim about the current state of the code ("已按 ADR-0037 退役") that is directly contradicted by the live code. `coach_daily` still chains into `ReviewPlanTask` every night, creating the "dual-channel" anti-pattern ADR-0037 was meant to eliminate.
- **建议**: Retire `review_plan` per ADR-0037: deregister from `practice/manifest.ts`, remove the `boss.send('review_plan')` call from `coach_daily.ts`, delete `review_plan.ts` handler and its tests; migrate the "know what to practice tonight" intent into the unified `composeDailyStream` / softmax-mfi selection engine. Or document explicitly why retirement is deferred (amend ADR-0039 §1 note to reflect that the job is still live pending the merge engine landing).
- Linear issue: title `Drift: ADR-0037 ↔ practice/manifest.ts + coach_daily.ts — review_plan job not retired`, labels `drift`

- **冲突**: The ADR documents this as a concrete schema addition (not phase-deferred), yet no migration or schema change has landed. The credit trickle-down path (RT2) is blocked on this column.

- Linear issue: title `Drift: ADR-0036 RT2 ↔ schema.ts — encompassing_weight column absent`, labels `drift`

#### [ADR-0039 §6] `ambient_context` not promoted to formal session-working-memory Postgres table
- **声明**: ADR-0039 §6 "把 `ambient_context` 从 `chat.ts` 私有入参升格为正式表，存当前面 / focused_entity / 上一轮练习结果 / 刚 dismiss 哪条"
- **代码**: `src/capabilities/copilot/server/chat.ts:168` — `ambient_context` remains a Zod-typed private request parameter; `src/db/schema.ts` has no `session_working_memory` or equivalent table; "刚 dismiss 哪条" dismiss-recall loop is not closed
- **冲突**: The ADR declares this a concrete decision ("两层正式契约" - formal contract), not a deferred item. Without the Postgres table, the "dismiss-recall" feedback loop (编排者下次不重复同类提议) and cross-surface session continuity are inoperative.
- **建议**: Migrate `ambient_context` to a `copilot_session_context` or `session_working_memory` Postgres table (subject-keyed or session-keyed); add write path on dismiss/advance events; thread into D14 orchestration reads. Or add an explicit deferral note in ADR-0039 §6 with a phase marker.
- Linear issue: title `Drift: ADR-0039 §6 ↔ chat.ts — ambient_context not promoted to Postgres table`, labels `drift`
