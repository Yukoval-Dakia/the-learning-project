### ❌ Contradicted (优先)

#### [ADR-0046 §"红线 amendments" / 首个 Rust-first 试点] A13 proper-scoring kernel shipped as a TS oracle, contradicting its own "no-TS-oracle" pilot designation
- **声明**: ADR-0046:41 "新数值 kernel 一律 Rust-first：无 TS oracle，直接 Rust 实现 + Rust-native 测试。不再制造新的双维护。" ADR-0046:43,85 explicitly names **A13 proper-scoring kernel 为「首个 Rust-first 试点」**（净新、边界干净、把它从「TS-first + 留 Rust follow-up」改为 Rust-first 单实现）.
- **代码**: `src/server/conjectures/scoring.ts:1-3` — the live proper-scoring kernel (brier/log-loss/skill_score_point) is a **TS implementation**, self-labeled in-line `// ADR-0046: proper-scoring = Rust-first single-source-of-truth. This TS is a PLACEHOLDER stub`. `crates/calibration-native/src/` contains only `lib.rs` — no `scoring.rs`, no Rust proper-scoring exists. The TS path is live and feeds production: ADR-0049's conjecture wire (`conjecture-accept.ts:203`) dispatches into it and `conjecture-scores.ts` reads its `prediction_score` events.
- **冲突**: ADR-0046's central rule ("无 TS oracle，直接 Rust 实现") was violated for the exact kernel the ADR names as its flagship pilot for that rule — it shipped as a live TS stub with no Rust counterpart. A later ADR (ADR-0049:6,43, 2026-07-04) quietly re-labels this "ADR-0046 Rust-deferred" in passing, but ADR-0046 itself was never amended to record the slip, so a reader of ADR-0046 alone sees an unqualified, now-false claim.
- **建议**: Amend ADR-0046 with a note (mirroring the ADR-0026 amendment pattern) recording that the A13 proper-scoring pilot shipped as an interim TS placeholder pending the Rust kernel — aligning it with ADR-0049's "Rust-deferred" language — or file the Rust port as a tracked follow-up so the flagship "no-TS-oracle" claim isn't left silently contradicted.
